# AI Sonic Resonance

> 把“听爽了”的瞬间，变成可互动、可分享、可共振的音乐事件。

**AI Sonic Resonance（AI 音浪共振器）** 是一个面向 QQ音乐普通听歌用户设计的 AI 音乐交互功能原型。

项目通过分析歌曲节奏、高潮段落、歌词情绪和用户共鸣点，将传统的静态播放器升级为可响应音乐与用户操作的沉浸式互动空间。

当用户听到鼓点密集、副歌开始、高音进入或情绪爆发等高能段落时，系统会在歌词中标记可互动的“共振点”。用户可以连续点击歌词，让歌词从轻微裂纹逐渐发展到最终爆裂，并触发页面升温、粒子掉落、进度条冲击和评论共振等视觉反馈。

歌曲结束后，系统还会生成一份具有游戏奖励感的能量结算结果，并允许用户进入歌词共振场，查看其他用户高亮的歌词、对应评论以及为歌词充能。

## 项目背景

当用户听到一首节奏感强、情绪浓烈或歌词击中自己的歌曲时，目前常见的表达方式主要是：

* 分享普通歌曲卡片
* 转发歌曲链接
* 截图歌词
* 在评论区留言
* 反复循环播放某个片段

这些方式很难充分表达用户在副歌爆发、鼓点进入、歌词击中或情绪转折时产生的强烈感受。

与此同时，传统播放器中的频谱、唱片旋转、镭射灯等视觉效果往往与具体歌词和歌曲结构缺少联系，无法真正回应用户在某个音乐瞬间产生的情绪。

AI 音浪共振器希望解决的问题是：

> 用户听到一首好歌时，缺少一种能够充分表达、互动和分享自己强烈感受的方式。

## 核心体验

项目的主要交互链路为：

```text
播放音乐
→ 歌词实时同步滚动
→ AI 标记高能互动点
→ 用户连续点击歌词
→ 裂纹逐步积累
→ 第五次点击触发完整爆裂
→ 页面进入音乐共振状态
→ 歌曲结束后生成能量结算
→ 进入歌词共振场
→ 查看评论并为歌词充能
```

## 核心功能

### 1. 时间轴歌词同步

项目使用带时间戳的 LRC 歌词或人工标注的歌词时间轴，而不是按照歌词总行数平均分配时间。

歌词同步以音频元素的实际播放时间为基准：

```javascript
audio.currentTime
```

支持：

* 当前歌词自动高亮
* 上下歌词动态切换
* 播放、暂停和继续播放
* 拖动进度条后重新定位歌词
* 歌曲重播后重置歌词状态
* 页面切换回来后重新校准
* 歌词整体时间偏移调节

---

### 2. AI 高能互动点

Demo 使用预设数据模拟 AI 对歌曲结构和情绪的识别。

支持的互动点包括：

* 鼓点密集
* 副歌开始
* 高音进入
* 节奏突变
* 情绪爆发
* 人声转折
* 乐器独奏
* 歌词高共鸣点

当歌曲接近互动点时，对应歌词会出现：

* 高亮
* 呼吸缩放
* 边缘发光
* 轻微浮动
* 粒子聚集
* 互动图标
* 进度条高能标记

---

### 3. 五段式歌词碎裂

歌词不会在第一次点击后立即消失。

每条可互动歌词拥有独立的点击次数和破坏状态。

#### 第一次点击

* 出现轻微裂纹
* 歌词短暂膨胀
* 播放轻微互动音效
* 原歌词保持完整

#### 第二次点击

* 裂纹逐渐扩展
* 文字局部轻微错位
* 出现少量碎屑

#### 第三次点击

* 歌词结构开始松动
* 少量粒子脱落
* 页面色温轻微升高
* 进度条出现冲击反馈

#### 第四次点击

* 歌词进入临界状态
* 裂纹开始发光
* 字块明显抖动
* 周围粒子聚集

页面提示：

> 共振已达临界点，再次点击释放音浪。

#### 第五次点击

* 歌词完整爆裂
* 碎片向页面边缘飞散
* 部分粒子掉落至播放器边框和进度条
* 页面出现冲击波和升温效果
* 播放完整共振音效
* 生成 AI 共振文案

爆裂特效使用独立粒子层实现，不永久删除原歌词数据。

---

### 4. 沉浸式歌词舞台

歌词位于页面视觉中央。

歌曲海报或封面以半透明方式置于歌词后方，并配合：

* 模糊
* 暗角
* 渐变遮罩
* 呼吸缩放
* 轻微空间位移
* 歌曲高潮色温变化

支持多种歌词动态模板：

* 阶梯式
* 流动式
* 融化式

普通播放状态保持克制，高潮状态释放强烈视觉反馈。

---

### 5. 页面升温与呼吸效果

当歌曲进入高潮或用户释放共振时，页面环境会发生渐进变化：

* 色温升高
* 饱和度增强
* 背景亮度变化
* 封面轻微放大
* 边框和进度条产生呼吸效果
* 页面随音乐能量收缩和扩张

高潮结束后，页面会逐渐恢复稳定状态，而不是突然切换。

---

### 6. 歌曲能量结算

歌曲播放结束后，系统进入能量结算页面。

分数使用类似金色老虎机或奖金滚动的动画呈现：

* 数字高速滚动
* 接近结果时逐渐减速
* 最终停止时产生冲击反馈
* 显示金属反光和粒子效果

示例指标：

* 实时能量
* 低频冲击
* 歌词共振
* 情绪峰值

这些数据用于演示产品交互，不代表真实生理或心理测量结果。

---

### 7. 歌词共振场

结算完成后，用户可以进入当前歌曲的歌词共振场。

其他用户高亮的歌词会以空间化形式交错展示。

歌词热度越高：

* 字体越大
* 亮度越高
* 越接近页面中心
* 粒子密度越高
* 呼吸效果越明显

页面使用粉色、蓝色和蓝绿色等赛博朋克霓虹色，但避免老式镭射和廉价霓虹效果。

---

### 8. 歌词评论互动

点击共振场中的高亮歌词后：

* 当前歌词被固定
* 其他歌词降低亮度
* 电流波或音频波从歌词边缘扩散
* 对应评论沿着波形逐条出现

示例评论：

> 这一句进来的时候真的被电到了。

> 我每次都会拖回来重听这一段。

> 鼓点一落下，整个人瞬间清醒。

> 戴耳机听这里完全不一样。

---

### 9. 歌词充能

用户可以为喜欢的歌词充能。

充能后会产生：

* 按钮反馈
* 互动音效
* 能量流向歌词
* 歌词短暂放大或变亮
* 充能数量增加
* 能量波扩散

产品逻辑为：

```text
用户为歌词充能
→ 系统记录歌词共振热度
→ 热度达到阈值
→ 歌词进入热门共振点候选
→ 后续用户播放时优先获得高亮提示
```

当前 Demo 使用本地状态或会话状态模拟用户充能数据。

## AI 能力说明

当前 Demo 使用预设数据模拟以下 AI 能力：

### 音乐结构识别

识别歌曲中的：

* 前奏
* 主歌
* 副歌
* 高潮
* 间奏
* 情绪转折

### 节奏与能量分析

识别：

* 鼓点密度
* 节奏强度
* 速度变化
* 能量峰值

### 歌词情绪识别

识别歌词的情绪类型，例如：

* 高燃
* 治愈
* 压抑
* 释放
* 浪漫
* 失重
* 破防

### 评论语义聚类

将评论与具体歌词、时间点和情绪类别建立关联。

### 共振文案生成

根据歌曲段落、歌词内容和用户操作生成个性化表达。

示例：

> 这一秒不是副歌，是我的精神过载点。

> 鼓点落下的瞬间，我也被重新启动。

> 这一句歌词不是唱出来的，是撞进来的。

## 部署注意事项

项目代码内部只使用相对路径，不依赖本地 Windows 绝对路径。

推荐使用：

```python
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent
AUDIO_PATH = BASE_DIR / "assets" / "audio" / "demo_track.mp3"
```

部署前请检查：

* 音频文件已经上传到 GitHub
* LRC 或歌词 JSON 文件已经上传
* 图片和音效文件已经上传
* 文件名大小写完全一致
* `requirements.txt` 包含全部必要依赖
* 代码中不存在本地绝对路径
* 代码中不存在 API Key 或其他密钥
* 应用不依赖本地启动的第二个服务

## 演示提示

为了让歌词顺利完成五次点击，Demo 可以为高能歌词设置适当的互动时间窗口。

建议：

* 高能歌词持续显示 5—8 秒
* 两次有效点击间隔约 150—250ms
* 第四次点击后出现临界提示
* 第五次点击与歌曲高潮同步
* 演示时避免拖动进度条跨过互动窗口

## 歌词文件格式

项目优先使用 LRC 格式歌词。

示例：

```lrc
[00:12.40]第一句歌词
[00:16.85]第二句歌词
[00:21.30]第三句歌词
```

项目会将其解析为：

```json
[
  {
    "id": "line-1",
    "startTime": 12.4,
    "endTime": 16.85,
    "text": "第一句歌词"
  },
  {
    "id": "line-2",
    "startTime": 16.85,
    "endTime": 21.3,
    "text": "第二句歌词"
  }
]
```

如果音频是经过剪辑的版本，需要调整歌词整体偏移量。

## 当前限制

当前版本是 Hackathon 产品原型，存在以下限制：

1. AI 音乐分析结果主要使用预设数据模拟。
2. 暂未接入 QQ音乐真实歌曲、歌词和评论接口。
3. 用户评论和充能数量为本地模拟数据。
4. 刷新页面后，部分互动状态可能被重置。
5. 只针对有限的 Demo 歌曲进行了歌词时间轴标注。
6. 音频、歌词和互动点需要保持版本一致。
7. 不同浏览器对自动播放和音频权限的限制不同。
8. 移动端复杂粒子效果可能自动降级。
9. Streamlit 重新运行可能导致前端组件重新初始化。
10. 当前分数不代表真实听觉、生理或心理测量结果。

## 项目价值

AI 音浪共振器并不试图重新设计完整的音乐播放器，也不依赖复杂的歌曲推荐系统。

它关注的是一个更具体的问题：

> 当一首歌真正击中用户时，播放器应该如何回应？

通过歌词互动、视觉共振、评论连接和好友充能，项目希望将听歌从被动播放转化为一种：

* 可表达
* 可参与
* 可分享
* 可传播
* 可共同体验

的音乐互动事件。
